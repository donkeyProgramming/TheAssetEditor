PackFile Handling manual checklist

Use this checklist when validating packfile behavior in AssetEditor builds. Mark each item as Pass/Fail and include notes where needed.

1. Load and create workflows

  1. Create New Project opens selected folder as editable project. [Pass/Fail]
  2. Open Project Folder opens existing folder project and sets editable. [Pass/Fail]
  3. Import Pack as Project extracts files and opens folder project. [Pass/Fail]
  4. Import Reference Pack loads as read-only container. [Pass/Fail]
  5. Open Game Packs loads game references without editable rights. [Pass/Fail]
  6. Open Recent Pack reopens expected item with correct mode. [Pass/Fail]

2. Editable pack behavior

  1. Set as Editable Pack updates active editable root. [Pass/Fail]
  2. Read-only roots cannot be set as editable. [Pass/Fail]
  3. Changes are applied only to editable/writable target. [Pass/Fail]

3. Tree edit operations

  1. Create Folder creates node and persists to container. [Pass/Fail]
  2. Import File adds file under selected folder/root. [Pass/Fail]
  3. Import Directory preserves folder structure. [Pass/Fail]
  4. Rename file updates path and remains openable. [Pass/Fail]
  5. Rename folder updates child paths correctly. [Pass/Fail]
  6. Delete file removes it from tree and container. [Pass/Fail]
  7. Delete folder removes full subtree. [Pass/Fail]
  8. Drag-drop move works inside same writable container only. [Pass/Fail]
  9. Copy to editable pack copies file/folder subtree from reference pack. [Pass/Fail]

4. Save and Save As

  1. Save prompts for path when system path is unknown. [Pass/Fail]
  2. Save uses existing path when known. [Pass/Fail]
  3. Save As always prompts for destination path. [Pass/Fail]
  4. After successful save, unsaved marker is cleared. [Pass/Fail]
  5. Saved .pack can be reopened and expected files exist. [Pass/Fail]

5. System folder specific behavior

  1. System folder project save outputs a .pack file and keeps source files. [Pass/Fail]
  2. project_ignore.json is created/updated on settings changes. [Pass/Fail]
  3. project_ignore.json is not included in saved .pack output. [Pass/Fail]
  4. Open/close/reopen project retains ignored-file settings. [Pass/Fail]
  5. External file create/delete/rename updates tree after watcher debounce. [Pass/Fail]

6. Version and ignore rules

  1. Selected project/container game version is reflected in saved pack header version. [Pass/Fail]
  2. Ignored files are excluded from serialization output. [Pass/Fail]
  3. Ignore matching handles case and slash-direction normalization. [Pass/Fail]

7. Error handling and resilience

  1. Save fails with clear error if output file is locked. [Pass/Fail]
  2. After lock release, save succeeds without restarting. [Pass/Fail]
  3. Read-only edit attempts are blocked with user feedback. [Pass/Fail]
  4. Closing pack root removes it from tree and active set. [Pass/Fail]

8. Export and external open tools

  1. Export to system folder writes selected root/folder/file output correctly. [Pass/Fail]
  2. Open in HxD / Notepad++ creates temp file and launches tool (if installed). [Pass/Fail]
