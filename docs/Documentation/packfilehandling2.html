PackFile handling (updated)

This page describes how packfiles are used in AssetEditor today. It covers normal .pack files, game reference packs, and system folder projects.

Pack types

  • Editable pack - The pack/project you are currently editing.
  • Reference pack - Read-only pack loaded for lookup/copy.
  • Game packs - Read-only CA pack collection for the selected game.
  • System folder project - A folder treated like an editable pack project.

Open or create content

Common entry points:

  • File => Create New Project - Create/open a system folder as editable project.
  • File => Open Project Folder - Open existing folder project.
  • File => Import Pack as Project - Extract .pack to folder and open it as project.
  • File => Import Reference Pack - Load a .pack as read-only reference.
  • File => Open Game Packs - Load all game packs for a game as read-only.
  • File => Open Recent Pack - Reopen recent pack/project.

Main editable pack

AssetEditor always has one editable pack/project at a time. To change it, right click a writable root and pick Set as Editable Pack.

PackFile explorer actions

Right click menus include:

  • Import File and Import Directory
  • Create Folder
  • Rename and Delete
  • Copy to editable pack (when source is another loaded pack)
  • Export to system folder
  • Open in HxD/Notepad++ for file inspection

Drag and drop can move files inside the same writable pack/project.

Saving

  • Save - Uses existing path if available. Otherwise asks for a .pack path.
  • Save As - Always asks for target .pack path.
  • System folder project save - Creates/updates a .pack file output, does not replace your source folder files.

Unsaved changes are marked in the tree and cleared when save completes.

System folder project settings

System folder projects store settings in project_ignore.json in the project root.

  • Selected game version for serialization
  • Ignored files list for serialization

The settings file itself is auto-excluded from saved pack content.

Known limitations and gotchas

  • Read-only containers (game/reference packs) cannot be edited directly.
  • If output file is locked by another process, save fails until the lock is released.
  • Ignore path matching supports case + slash-direction normalization, but not duplicate/leading separator canonicalization.
  • System folder save writes a .pack output path and keeps source project files unchanged.

Troubleshooting

  • Cannot edit - Verify selected root is writable and set as editable.
  • Save fails - Check file lock (game/tool/antivirus) on target file.
  • Wrong output version - Check project game version setting.
  • Missing file after save - Check ignored files list and normalized path.
